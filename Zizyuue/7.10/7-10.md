# 2026-07-10 学习总结：robomimic 上的 Behavior Cloning

## 1. 今日目标

今天开始从之前的二维玩具 Behavior Cloning 任务，过渡到标准机器人操作数据集。

本次使用：

- 框架：`robomimic`
- 任务：`Lift`
- 数据类型：`PH`（Proficient Human，熟练人类示范）
- 数据形式：`low_dim`
- 数据文件：`datasets/lift/ph/low_dim_v15.hdf5`

当前任务本质是：

> 根据机械臂、夹爪和方块的当前状态，预测专家在当前时刻采取的 7 维控制动作。

---

## 2. robomimic 数据集认识

`robomimic` 不只是一个数据集，而是一套机器人模仿学习与离线强化学习框架。

本次使用的 `Lift-PH low_dim` 数据包含：

- 200 条人类成功示范轨迹
- 总计 9666 个时间步
- 每条轨迹长度不同
- 最短轨迹：36 步
- 最长轨迹：64 步
- 平均轨迹长度：48.33 步

每条轨迹包含：

```text
actions
dones
next_obs
obs
rewards
states
```

含义：

- `obs[t]`：第 `t` 步执行动作前的观察
- `actions[t]`：专家在第 `t` 步采取的动作
- `next_obs[t]`：动作执行后的下一状态
- `rewards[t]`：任务奖励
- `dones[t]`：是否达到终止或成功状态
- `states[t]`：MuJoCo 仿真器内部完整状态

最基础的 Behavior Cloning 只学习：

```text
obs[t] -> action[t]
```

---

## 3. observation 字段

本次选择了四类 observation：

```python
OBS_KEYS = [
    "object",
    "robot0_eef_pos",
    "robot0_eef_quat",
    "robot0_gripper_qpos",
]
```

维度分别为：

```text
object                 10维
robot0_eef_pos          3维
robot0_eef_quat         4维
robot0_gripper_qpos     2维
```

拼接后：

```text
observation 维度 = 10 + 3 + 4 + 2 = 19
```

### object 的 10 维

大致由以下信息组成：

```text
方块位置              3维
方块姿态四元数        4维
方块相对夹爪的位置    3维
```

### robot0_eef_pos

机械臂末端执行器位置：

```text
[x, y, z]
```

### robot0_eef_quat

机械臂末端执行器姿态四元数，共 4 维。

### robot0_gripper_qpos

夹爪两个手指关节的位置，共 2 维。

---

## 4. action 字段

每个动作是 7 维：

```text
前3维：末端位置控制
中间3维：末端旋转控制
最后1维：夹爪控制
```

可近似写为：

```text
[dx, dy, dz, drx, dry, drz, gripper]
```

动作值通常位于 `[-1, 1]` 附近。

需要注意：

- 前 6 维是连续控制量
- 最后 1 维夹爪动作接近离散决策，通常接近 `-1` 或 `1`

## 5. 数据读取与整理

成功将 200 条不等长轨迹整理成单步训练样本：

```text
observations shape: (9666, 19)
actions shape:      (9666, 7)
```

单个训练样本为：

```text
19维 observation -> 7维 action
```

数据读取核心逻辑：

```python
obs_parts = []

for key in OBS_KEYS:
    obs_parts.append(demo["obs"][key][:])

observations = np.concatenate(
    obs_parts,
    axis=-1
)

actions = demo["actions"][:]
```

同一条轨迹内部沿特征维拼接：

```text
(T, 10) + (T, 3) + (T, 4) + (T, 2)
-> (T, 19)
```

不同轨迹之间沿时间维拼接：

```text
demo_0
demo_1
...
demo_199
-> (9666, 19)
```

## 6. 训练集与验证集划分

没有直接随机切分 9666 个时间步，而是按完整轨迹划分。

划分结果：

```text
训练轨迹：160 条
验证轨迹：40 条
```

对应时间步：

```text
训练集：
observations shape: (7741, 19)
actions shape:      (7741, 7)

验证集：
observations shape: (1925, 19)
actions shape:      (1925, 7)
```

训练集和验证集没有重复轨迹：

```text
重复轨迹：set()
```

按轨迹划分可以避免同一条轨迹中的相邻状态同时进入训练集和验证集，从而减少数据泄漏。

## 8. 当前 MLP 网络

当前网络结构：

```text
19 -> 256 -> 512 -> 256 -> 128 -> 7
```

对应代码结构：

```python
nn.Linear(19, 256)
nn.ReLU()

nn.Linear(256, 512)
nn.ReLU()

nn.Linear(512, 256)
nn.ReLU()

nn.Linear(256, 128)
nn.ReLU()

nn.Linear(128, 7)
```

网络经过 4 个隐藏层，隐藏层宽度变化为：

```text
×2 -> ×2 -> ÷2 -> ÷2
```

最后一层没有使用 `ReLU`，因为动作中包含负数。

## 9. 训练结果

训练使用：

- 损失函数：`MSELoss`
- 优化器：`Adam`
- 学习率：`5e-4`
- 最佳模型按验证损失保存

最佳结果：

```text
最佳 Epoch：376
最佳验证损失：0.025762778
```

训练曲线表现：

```text
训练损失持续下降并接近 0
验证损失先下降，随后逐渐上升
```

说明当前模型存在明显过拟合。

训练 5000 个 epoch 明显过多，后续应加入：

- Early Stopping
- observation 标准化
- 适当减小模型
- Dropout 或 LayerNorm

## 11. 当前主要问题

### 11.1 明显过拟合

训练损失接近 0，但验证损失在早期达到最低后逐渐上升。

说明网络逐渐记住训练轨迹，但对未见轨迹的泛化能力下降。

### 11.2 夹爪动作误差最大

`gripper` 的 MSE 明显高于其他动作维度：

```text
gripper MSE: 0.10759988
```

原因是夹爪动作通常接近：

```text
-1 或 1
```

它更接近二分类问题，而不是普通连续回归问题。

当前全部使用 MSE 时，模型在不确定的位置容易预测出：

```text
-0.15
0.68
0.84
```

这样的中间值。

### 11.3 当前仍是单步模型

虽然数据原本来自序列轨迹，但当前已经被拆成单步样本：

```text
obs[t] -> action[t]
```

模型看不到历史状态，因此不知道当前处于：

- 靠近阶段
- 抓取阶段
- 闭合夹爪阶段
- 抬升阶段

所以当前不是序列推理，也没有注意力机制。

## 13. 下一步计划

下一次可以按以下顺序继续：

### 第一步：重新训练一个更稳定的 baseline

加入：

```text
observation 标准化
Early Stopping
较少的最大 epoch
保存标准化参数
```

### 第二步：双头网络

共享 backbone 后拆成两个输出头：

```text
motion head：预测前 6 维连续动作
gripper head：预测夹爪开闭
```

损失：

```text
前6维：MSELoss
夹爪：BCEWithLogitsLoss
```

### 第三步：闭环 rollout

把策略放进 robosuite 环境：

```text
环境 observation
-> 模型预测 action
-> 环境执行 action
-> 返回下一 observation
-> 循环
```

用成功率而不是仅用离线 MSE 判断模型效果。

### 第四步：序列模型

将 Dataset 改造成滑动窗口：

```text
过去若干步 observation
-> GRU / Transformer / Attention
-> 当前 action
```

这样模型才能利用历史信息判断抓取阶段。
